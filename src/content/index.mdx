# Snippets from Real JavaScript Interviews

This site collects the JavaScript patterns that repeatedly show up in frontend interviews and day-to-day product work.
The goal is not to memorize snippets. The goal is to read code, predict behavior, explain the trade-offs, and recognize the same pattern when it appears in a different form.

Most pages start with a small problem and then reveal the implementation. Copy the code when you need to, but try to reason through the output first. That is the habit that makes these examples useful in real work.

> Tip: when an example feels unclear, run it once and then step through it in a debugger. The breakpoint is usually more useful than another explanation.

## Predict the output

### What will this print?

```js filename="Promise Resolution Order" copy
const promise1 = Promise.resolve();
const promise2 = Promise.resolve();

promise1.then(() => console.log(1)).then(() => console.log(2));
promise2.then(() => console.log(3)).then(() => console.log(4));
```

### And how about this?

```js filename="Event Loop" copy
console.log("1");

setTimeout(function () {
  console.log("2");

  Promise.resolve().then(function () {
    console.log("3");
  });
}, 0);

Promise.resolve().then(function () {
  console.log("4");

  setTimeout(function () {
    console.log("5");
  }, 0);
});

requestAnimationFrame(function () {
  console.log("7");
});

console.log("6");
```
