# Queue and Stack Patterns in JavaScript

Queues and stacks are small abstractions, but they show up everywhere: schedulers, breadth-first search, undo history, parsers, and async workflows. The examples stay plain so the ordering rule is visible in each implementation.

## AsyncQueue: Processing Promises Sequentially

`AsyncQueue` stores functions, not already-running promises. That distinction matters: each task starts only after the previous task has resolved, so the result order follows the queue order even when later tasks would finish faster on their own.

```js filename="async queue" copy
class AsyncQueue {
  constructor() {
    this.queue = [];
  }

  enqueue(task) {
    if (typeof task !== "function") {
      throw new Error("Task must be a function returning a promise");
    }
    this.queue.push(task);
  }

  async process() {
    const results = [];

    for (const task of this.queue) {
      const result = await task();
      results.push(result);
    }

    return results;
  }
}

const queue = new AsyncQueue();
queue.enqueue(
  () =>
    new Promise((resolve) => {
      setTimeout(() => resolve("Task 1"), 1000);
    }),
);
queue.enqueue(
  () =>
    new Promise((resolve) => {
      setTimeout(() => resolve("Task 2"), 500);
    }),
);

queue.process().then((results) => console.log(results));
// ['Task 1', 'Task 2']
```

## Queue Implementation: First-In, First-Out (FIFO)

A queue follows **FIFO**: first in, first out. New values are added to the back, and `dequeue()` removes from the front. This is the same ordering used by task queues and breadth-first search.

```js filename="queue" copy
class Queue {
  constructor() {
    this.collection = [];
  }

  enqueue(element) {
    this.collection.push(element);
    return this;
  }
  dequeue() {
    this.collection.shift();
    return this;
  }
  front() {
    return this.collection[0];
  }
  get size() {
    return this.collection.length;
  }
  isEmpty() {
    return this.collection.length === 0;
  }
}

const queue = new Queue();
queue.enqueue("one").enqueue("two");
// [ 'one', 'two' ]
queue.dequeue(); // [ 'two' ]
queue.enqueue("three").enqueue("four");
// [ 'two', 'three', 'four' ]
queue.front(); // two
queue.size; // 3
queue.isEmpty(); // false
```

## Using a Stack to Check for Palindromes

A stack reverses order because the last value pushed is the first one popped. The palindrome check uses that property to rebuild the word backward and compare it with the original.

```js filename="stack palindrome" copy
function isPalindrome(word) {
  const letters = []; // this is our stack
  let reversedWord = "";

  // put letters of word into stack
  for (let i = 0; i < word.length; i++) {
    letters.push(word[i]);
  }

  // pop off the stack in reverse order
  for (let i = 0; i < word.length; i++) {
    reversedWord += letters.pop();
  }

  return reversedWord === word;
}

console.log(isPalindrome("racecar")); // true
console.log(isPalindrome("cheatsheet")); // false
```

## Stack Implementation: Last-In, First-Out (LIFO)

A stack follows **LIFO**: last in, first out. `push()` adds to the top, `pop()` removes from the top, and `peek()` reads the current top without removing it.

Useful for:

- Undo functionality
- Recursion
- Reversing data

```js filename="stack" copy
class Stack {
  constructor() {
    this.storage = [];
  }

  push(value) {
    this.storage.push(value);
    return this;
  }

  pop() {
    this.storage.pop();
    return this;
  }

  size() {
    return this.storage.length;
  }

  peek() {
    return this.storage[this.storage.length - 1];
  }
}

const myStack = new Stack();

myStack.push(1).push(2).push(3);
console.log(myStack.peek()); // 3
myStack.pop();
console.log(myStack.peek()); // 2
myStack.push("js-cheatsheet");
console.log(myStack.size()); // 3
console.log(myStack.peek()); // js-cheatsheet
```
